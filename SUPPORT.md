# Support

## Community Support

Use GitHub issues for bugs, documentation gaps, and feature requests.

Before opening an issue:

- search existing issues;
- verify the behavior against the relevant RFC or registry documentation;
- remove credentials and sensitive values from logs or XML;
- include the SDK version or commit.

## Security Issues

Do not open public issues for suspected vulnerabilities. See [SECURITY.md](SECURITY.md).

## Registry Integration Help

Registry compatibility questions should include:

- registry name and OT&E/production environment, if shareable;
- supported namespaces from the greeting;
- redacted command XML;
- redacted response XML;
- relevant registry documentation.
